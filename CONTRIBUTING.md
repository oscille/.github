# Contributing to Oscille

Thanks for contributing. These guidelines apply across Oscille repositories unless a repo states otherwise.

## Getting started

1. Branch from `main`, or fork and then branch.
2. Install the repository's dependencies.
3. Name the branch `feature/<desc>`, `fix/<desc>`, or `chore/<desc>`.

## Making changes

- Follow the conventions in [`conventions/`](conventions/).
- Keep pull requests small and focused.
- Write commit messages as [Conventional Commits](https://www.conventionalcommits.org/).
- Make sure lint, typecheck, and tests pass.
- Update the docs when behavior changes.

## Pull requests

- Fill out the pull request template.
- Make sure CI is green before requesting review.
- Pull requests squash-merge into `main`.

## Reporting bugs and requesting features

Open an issue using the appropriate template. For a security issue, follow [`SECURITY.md`](SECURITY.md) rather than filing a public issue.
