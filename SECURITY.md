# Security Policy

## Reporting a vulnerability

Please report security vulnerabilities **privately**. Do not open a public issue.

- Use [GitHub private vulnerability reporting](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing-information-about-vulnerabilities/privately-reporting-a-security-vulnerability)
  on the affected repository, **or**
- Email **security@oscille.com**

Please include steps to reproduce, affected versions, and impact if known.

## What to expect

- We'll acknowledge your report as soon as we can.
- We'll keep you updated on remediation progress.
- We'll credit you (if you wish) once a fix is released.

## Supported versions

Unless a repository states otherwise, only the latest released version receives
security updates.
